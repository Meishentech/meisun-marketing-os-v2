## 已複查問題：行銷案新增後 UI 沒有可靠顯示新增列

複查方式：不只讀程式碼，直接在瀏覽器實測完整流程，攔截並記錄每個 API 呼叫的時間戳記。

**判定：不是資料或邏輯錯誤，是「有真實延遲、但延遲期間畫面沒有任何回饋」造成的體驗缺口。**

實測時間軸（送出表單起算）：
- `~0-300ms`：`nextCampaignSortOrder()` 前置查詢（行銷案新增特有的額外步驟）
- `~300-600ms`：`POST marketing_campaigns` 完成，modal 立即關閉
- `~600-1030ms`：`loadExistingData()` 平行重撈全站 20 幾張表，資料才真正換新、畫面才重繪

新行銷案確實正確寫入且出現在列表，全程 console 無錯誤。問題在於 modal 在 POST 完成當下（~600ms）就關閉了，但畫面實際刷新要再等 400-600ms，這段期間完全沒有 loading／刷新中的視覺提示——程式碼裡沒有任何地方處理 `state.dataStatus === "loading"` 這個狀態。這個組合容易讓自動化工具「點完立刻檢查」誤判成「沒反應」。

**不擋驗收。** 可選的體驗優化（非必要）：讓 modal 在 `loadExistingData()` 真正完成後才關閉，或按鈕顯示「建立中...」直到流程跑完。此機制是全站共用（每次新增/編輯都會觸發整個 `loadExistingData()`），不是行銷案專屬，如果要改建議另開小 batch。
